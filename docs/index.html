Claude Voice

Claude Code, ahora te habla

Lee en voz alta las respuestas, preguntas y pasos a seguir de Claude Code, con la voz del sistema de macOS. Sin dependencias, sin tokens, sin internet.

02

Tutorial completo

2.1Instalar

Necesitas macOS, Claude Code y python3 (viene con las Command Line Tools: xcode-select --install).

$ git clone https://github.com/lowlufi/claude-voice.git $ cd claude-voice $ ./install.sh Hooks registrados en ~/.claude/settings.json Comando instalado: /opt/homebrew/bin/voz $ voz test 🔊 “Hola. Soy Claude, y a partir de ahora puedes escucharme.”

Eso es todo: toda sesión nueva de Claude Code te hablará sola, en todos tus proyectos. Correr el instalador dos veces no duplica ni rompe nada.

2.2Activar y desactivar

La voz nace activada. Tres niveles de control, del más suave al más definitivo:

voz callateCorta lo que esté diciendo ahora mismo. También se calla sola en cuanto envías un nuevo prompt.
voz offSilencio total en todas las sesiones, sin desinstalar nada.
voz onVuelve a hablar desde la siguiente respuesta.
voz estadoMuestra si está activa, con qué voz, a qué velocidad y en qué modo.
TIP — también puedes pedírselo a Claude en la propia sesión: “voz off”, y él ejecuta el comando por ti.

2.3Cambiar la voz y el idioma

Por defecto elige una voz en español automáticamente (Paulina, es-MX). Para listar las voces de tu Mac:

$ say -v '?' Paulina es_MX # ¡Hola! Me llamo Paulina. Mónica es_ES # ¡Hola! Me llamo Mónica. Samantha en_US # Hello! My name is Samantha.

Y para cambiarla — vale cualquier voz, de cualquier idioma:

$ voz usar Mónica # español de España $ voz usar Samantha # inglés $ voz usar auto # volver a la selección automática
VOCES PREMIUM — las voces "Enhanced" suenan mucho más naturales y son gratis: Ajustes del Sistema → Accesibilidad → Contenido hablado → Voz del sistema → Gestionar voces… → descarga Paulina (Enhanced). claude-voice la usará automáticamente.

2.4Modos y velocidad

voz modo fullToda la respuesta, limpia: sin código ni tablas, listas con pausas. Es el default.
voz modo briefSolo lo esencial: primeras frases y cualquier pregunta que te haga.
voz modo summaryResumen de 2-3 frases generado con Haiku. Único modo que usa tokens.
voz velocidad 210Palabras por minuto (default 185).

La configuración vive en config.json y los cambios aplican de inmediato.

2.5Referencia de comandos

$ voz test # prueba de voz $ voz callate # cortar la lectura en curso $ voz off / on # silenciar del todo / reactivar $ voz modo brief # full | brief | summary | off $ voz velocidad 200 # palabras por minuto $ voz usar Mónica # cambiar de voz (o: auto) $ voz estado # ver configuración actual

2.6Desinstalar

Sin residuos: quita los hooks, el comando voz y los archivos de estado. Tu configuración de Claude Code queda intacta.

$ ./uninstall.sh Hooks eliminados de ~/.claude/settings.json Listo.
03

Especificaciones

MOTORSintetizador say de macOS — 100% local, funciona sin internet.
COSTOCero tokens, cero APIs. (Excepción: modo summary, opcional y apagado por defecto.)
LECTURALimpia el markdown: omite código y tablas, acorta rutas, lee listas con pausas.
AVISOS“En mi-proyecto: Claude necesita tu permiso para usar Bash.” Con el nombre del proyecto.
SILENCIOSe calla al enviar un prompt; los avisos de inactividad nunca interrumpen una lectura.
MULTI-SESIÓNUna sola voz a la vez (candado flock); la respuesta más reciente gana.
ACCESIBILIDADAliada para desarrolladores ciegos o con baja visión; complementa a VoiceOver.
LICENCIAMIT — open source, con tests incluidos.
04

Preguntas frecuentes

¿Funciona solo con Claude Code o también con otros agentes?
Out-of-the-box está hecha para Claude Code (usa su sistema de hooks). Pero el corazón es un script independiente: cualquier herramienta capaz de ejecutar un comando puede hablarte con echo '{"last_assistant_message":"tu texto"}' | claude_voice.py stop. Si tu agente tiene hooks o eventos, adaptarlo toma minutos — ports bienvenidos en los issues.
¿Gasta tokens o dinero?
No. Sintetizador local de macOS: cero tokens, cero APIs, sin internet. La única excepción es el modo opcional summary, apagado por defecto.
¿En qué idiomas habla?
En el que quieras: cualquier voz de say -v '?'. Por defecto español (Paulina); voz usar Samantha la pasa a inglés, voz usar Amélie a francés.
¿Solo macOS?
Por ahora sí — depende del comando say. El port a Linux (espeak-ng, piper) o Windows es la contribución más pedida.
¿Lee el código también?
No, a propósito: el código dictado es insufrible. Escuchas el análisis, los hallazgos, las preguntas y los pasos; el código lo ves en la terminal.
¿Y con varias sesiones abiertas?
Todas te hablan, pero nunca a la vez: un candado garantiza una sola voz y los avisos dicen de qué proyecto vienen.
¿Cómo la apago rápido si estoy en una llamada?
voz callate (corta ya) o voz off (silencio hasta nuevo aviso). Y al escribir un prompt se calla sola.
